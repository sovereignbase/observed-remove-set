This document specifies the behavior of the @sovereignbase/observed-remove-set package as implemented by its current source. It defines the package's data model, mutation semantics, event model, error conditions, and the algorithms that govern construction, append, remove, clear, merge, and snapshot production.

This specification is an unofficial document maintained alongside the package implementation. It is intended to make the package's behavior explicit and auditable.

The key words MUST, MUST NOT, SHOULD, SHOULD NOT, and MAY in this document are to be interpreted as described in BCP 14 [[RFC2119]] [[RFC8174]] when, and only when, they appear in all capitals.

A conforming implementation of this specification MUST expose the behavior defined in the algorithms below. A conforming use of the API MUST treat the package-maintained state, including the size property, as implementation-owned state.

Introduction

An observed-remove set, or OR-Set, is a set CRDT in which additions are tracked by stable identifiers and removals are represented as tombstones. This package uses UUID version 7 values as entry identifiers and exposes a small JavaScript API centered around snapshots and merge operations.

The package is designed for membership-style data and static metadata. It is not a field-level CRDT for deep object mutation. The implementation freezes each stored entry shallowly, but nested object mutation remains outside the package's replication semantics.

This specification describes the package's current semantics, including cases where malformed top-level snapshots throw while malformed individual tomb or item entries are ignored.

Terminology

A valid UUIDv7 string is a string that is a valid UUID and whose version field is 7 as defined by [[RFC9562]].

An entry is an object that has a __uuidv7 property whose value is intended to be a valid UUIDv7 string.

A live entry is an entry whose identifier is present in the implementation's item store and absent from its tomb set.

A tombstone is a UUIDv7 string recorded in the implementation's tomb set.

A snapshot is an object with an items array and a tombs array.

A well-formed snapshot is a value that is an Object and whose items member is an Array and whose tombs member is an Array.

A malformed snapshot is any value that is not a well-formed snapshot.

A no-op mutation is an API call that does not change the live item store and does not change the tomb set.

Data Model

Each OR-Set instance maintains two pieces of internal state:

  • an item store keyed by UUIDv7 string and containing frozen entry objects; and
  • a tomb set containing UUIDv7 strings.

The size property reflects the number of live entries in the item store.

The implementation MUST treat the first accepted live entry for a given UUIDv7 as authoritative. Later duplicates with the same UUIDv7 MUST be ignored unless the UUID is absent from the live item store and not tombstoned.

Invalid individual tombs and invalid individual items MUST be ignored. Only malformed top-level snapshots are fatal.

};

dictionary ORSetSnapshot {
  sequence<any> items = [];
  sequence<DOMString> tombs = [];
};

dictionary ORSetMergeResult {
  sequence<DOMString> removals = [];
  sequence<any> additions = [];
};

The package's runtime surface exports the ORSet constructor. The package's type surface additionally exposes the ORSetErrorCode union, currently containing only BAD_SNAPSHOT.

Event Model

Implementations MUST support listener registration through addEventListener() and removeEventListener().

Implementations MUST dispatch the following events:

delta event
A CustomEvent whose detail is an ORSetSnapshot-shaped delta payload. It is dispatched after successful append(), remove(), and clear() operations. It is not dispatched by merge().
snapshot event
A CustomEvent whose detail is the full current snapshot after a state-changing mutation.
merge event
A CustomEvent whose detail is an ORSetMergeResult. It is dispatched only when merge() changes state.

A no-op mutation MUST NOT dispatch any event.

Error Model

When this specification requires throwing an ORSetError, the implementation MUST throw an Error-derived object whose name is ORSetError and whose code is BAD_SNAPSHOT, a member of ORSetErrorCode.

Constructor input and merge input MUST throw ORSetError when the provided snapshot is malformed snapshot. Invalid tombs and invalid items inside an otherwise well-formed snapshot MUST be ignored and MUST NOT throw.

Algorithms

Constructor

The ORSet(snapshot) constructor initializes a new OR-Set instance.

  1. Set size to 0.
  2. Initialize the item store to an empty object and the tomb set to an empty set.
  3. If snapshot is undefined, then return.
  4. If snapshot is a malformed snapshot, then throw ORSetError.
  5. For each tomb in snapshot's tombs array:
    1. If tomb is not a valid UUIDv7 string, continue.
    2. Add tomb to the tomb set.
  6. For each item in snapshot's items array:
    1. Let v7 be item's __uuidv7 member.
    2. If v7 is not a valid UUIDv7 string, continue.
    3. If the tomb set contains v7, continue.
    4. If the item store already has its own property named v7, continue.
    5. Store Object.freeze(item) in the item store at v7.
    6. Increase size by 1.

has()

The has(value) method returns whether the item store has its own property whose name is value.__uuidv7.

append()

  1. Let v7 be entry.__uuidv7.
  2. If v7 is a valid UUIDv7 string and the item store already has its own property named v7, then return.
  3. If v7 is a valid UUIDv7 string and the tomb set does not contain v7, then let stored entry be Object.freeze(entry).
  4. Otherwise, let stored entry be Object.freeze({...entry, __uuidv7: new UUIDv7}), where new UUIDv7 is a freshly generated valid UUIDv7 string.
  5. Let nextV7 be stored entry's __uuidv7.
  6. Store stored entry in the item store at nextV7.
  7. Increase size by 1.
  8. Dispatch a delta event whose detail contains items: [stored entry] and tombs: [].
  9. Dispatch a snapshot event whose detail is the result of running snapshot().

clear()

  1. If size is 0, then return.
  2. Let egress tombs be a new empty list.
  3. For each own property name v7 of the item store:
    1. Add v7 to the tomb set.
    2. Delete the item store entry at v7.
    3. Append v7 to egress tombs.
  4. Set size to 0.
  5. Dispatch a delta event whose detail contains items: [] and tombs: egress tombs.
  6. Dispatch a snapshot event whose detail is the result of running snapshot().

remove()

  1. Let v7 be entry.__uuidv7.
  2. If v7 is not a valid UUIDv7 string, then return.
  3. Let had item be whether the item store has its own property named v7.
  4. Let had tomb be whether the tomb set contains v7.
  5. If had item is false and had tomb is true, then return.
  6. Add v7 to the tomb set.
  7. Delete the item store entry at v7.
  8. If had item is true, decrease size by 1.
  9. Dispatch a delta event whose detail contains items: [] and tombs: [v7].
  10. Dispatch a snapshot event whose detail is the result of running snapshot().

This algorithm intentionally permits recording a tombstone for a valid UUIDv7 that does not currently correspond to a live item.

values()

The values() method returns a new array containing the current live entries in the item store's enumeration order.

merge()

  1. If ingress is a malformed snapshot, then throw ORSetError.
  2. Let additions be a new empty list.
  3. Let removals be a new empty list.
  4. For each tomb in ingress's tombs array:
    1. If the tomb set already contains tomb, continue.
    2. If tomb is not a valid UUIDv7 string, continue.
    3. Let had item be whether the item store has its own property named tomb.
    4. Add tomb to the tomb set.
    5. Delete the item store entry at tomb.
    6. If had item is true, decrease size by 1.
    7. Append tomb to removals.
  5. For each entry in ingress's items array:
    1. Let v7 be entry's __uuidv7 member.
    2. If v7 is not a valid UUIDv7 string, continue.
    3. If the tomb set contains v7, continue.
    4. If the item store already has its own property named v7, continue.
    5. Store Object.freeze(entry) in the item store at v7.
    6. Increase size by 1.
    7. Append entry to additions.
  6. If additions is empty and removals is empty, then return.
  7. Dispatch a merge event whose detail contains additions and removals.
  8. Dispatch a snapshot event whose detail is the result of running snapshot().

Tombs are processed before ingress items. As a result, an ingress item whose UUID is tombstoned by the same merge operation is not re-added.

snapshot()

  1. Return a new object whose items member is a new array containing the item store's values and whose tombs member is a new array containing the tomb set's values.

addEventListener() and removeEventListener()

These methods forward listener registration and listener removal, respectively, to the instance's internal event target using the provided type, listener, and options values.

Conforming implementations MAY accept listener types other than delta, snapshot, and merge, but this specification only defines dispatch behavior for those three event types.

Behavioral Notes

  • The package performs shallow freezing of stored entries. Nested values remain mutable and are not replicated with object-field CRDT semantics.
  • append() preserves a caller-supplied UUIDv7 only when that UUID is valid, not currently live, and not tombstoned.
  • merge() emits merge and snapshot events when it changes state, but it does not emit a delta event.
  • values() and snapshot() return new arrays, but the entry objects inside those arrays are the frozen stored entries.
